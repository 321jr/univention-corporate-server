UCS Erstkonfiguration

Herzlich Willkommen

Die UCS Erstkonfiguration führt Sie durch die initiale Konfiguration, die notwendig ist, damit dieses neu erzeugte System verwendet werden kann.

Dieser Vorgang beinhaltet alle notwendigen Einstellungen, damit das System einer Domäne beitreten kann oder eine neue erzeugt wird. Darüber hinaus können weitere Einstellungen wie z.B. die Software-Ausstattung vorgenommen werden.


Die Erstkonfiguration des Systems ist aufgeteilt in die folgenden Schritte:

  1. Die Systemrolle des Systems kann geändert werden, solange es noch nicht der UCS-Domäne beigetreten ist.
  2. Die Spracheinstellungen beinhalten alle sprachrelevanten Einstellungsmöglichkeiten wie Zeitzone, Tastaturlayout und Systemsprache.
  3. Die Basis-Einstellungen legen grundlegende Einstellungen wie Rechner- und Domänenname, LDAP-Basis, Windows-Domänename und das Passwort für die Systemadministration (root) fest.
  4. In den Netzwerkeinstellungen können IP-Adressen (IPv4 und IPv6) sowie DNS-Server, Gateways und HTTP-Proxy festgelegt werden.
  5. Nach Vergabe der Zertifikatseinstellungen wird ein neues Root-Zertifikat für die Domäne erstellt. Hinweis: Dieser Schritt ist nur auf Domänencontroller Master-Systemen notwendig.
  6. Über die Software-Einstellungen können bestimmte Software-Komponenten installiert und deinstalliert werden.

Während des Speichervorgangs wird auch der Domänenbeitritt durchgeführt. Dazu ist auf allen Systemrollen, außer Domänencontroller Master, die Eingabe von Benutzernamen und Passwort eines gültigen Domänenadministrator-Kontos notwendig. Auf dem Domänencontroller Master wird eine neue Domäne erzeugt. Nach dem erfolgreichen Domänenbeitritt wird der Einstellungsassistent beendet und der Boot-Vorgang fortgesetzt.

Werden die Einstellungen in einem Browser von einem externen System aus vorgenommen, ist im Anschluss an die Konfiguration eine Neuanmeldung an Univention Management Console erforderlich.

Die Erstkonfiguration kann jederzeit im UMC-Modul "Basiseinstellungen" geändert werden.
