Allgemein

UCS Appliance-Modus ermöglicht es, die Grundkonfiguration eines Univention Corporate Server-Systems nach dem Boot-Vorgang per Browser anzupassen. So können beispielsweise vorbereitete Images von Univention Corporate Server (UCS) kopiert oder für virtuelle Instanzen geklont und beim ersten Systemstart per Browser konfiguriert werden. Die Grundkonfiguration beinhaltet die notwendigen Einstellungen, damit das System einer Domäne beitreten kann und somit direkt einsatzfähig ist. Darüber hinaus können weitere Einstellungen wie z.B. die Software-Ausstattung vorgenommen werden.

Konfiguration

Die Grundkonfiguration des Systems ist aufgeteilt in die folgenden Schritte:

  1. Die Spracheinstellungen beinhalten alle sprachrelevanten Einstellungsmöglichkeiten wie Zeitzone, Tastaturlayout und Systemsprache.
  2. Die Basis-Einstellungen legen grundlegende Einstellungen wie Rechner- und Domänenname, LDAP-Basis, Windows-Domänename und Root-Passwort fest.
  3. In den Netzwerkeinstellungen können IP-Adressen (IPv4 und IPv6) sowie DNS-Server, Gateways und HTTP-Proxy festgelegt werden.
  4. Nach Vergabe der Zertifikatseinstellungen wird ein neues Root-Zertifikat für die Domäne erstellt. Hinweis: Dieser Schritt ist nur auf Domänencontroller Master-Systemen notwendig.
  5. Über die Software-Einstellungen können bestimmte Software-Komponenten installiert und deinstalliert werden.

Während des Speichervorgangs wird auch der Domänenbeitritt durchgeführt. Dazu ist auf allen Systemrollen, außer Domänencontroller Master, die Eingabe von Benutzernamen und Passwort eines gültigen Domänenadministrator-Kontos notwendig. Nach dem erfolgreichen Domänenbeitritt kann der Einstellungsassistent abgeschlossen und der Boot-Vorgang fortgesetzt werden.

Werden die Einstellungen in einem Browser von einem externen System aus vorgenommen, ist im Anschluss an die Konfiguration eine Neuanmeldung an Univention Management Console erforderlich.

Die Grundkonfiguration kann jederzeit im UMC-Modul "Basiseinstellungen" geändert werden.
