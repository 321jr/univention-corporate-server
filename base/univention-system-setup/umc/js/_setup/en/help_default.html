UCS initial configuration

Welcome

The UCS initial configuration will guide you through the configuration steps required to use this newly created system.

The process includes all necessary settings to join the system to a domain or to create a new one. Moreover, additional settings, such as software equipment, may be adjusted.


The initial configuration is divided into the following steps:

  1. If the system is not part of a domain yet, the system role may be changed.
  2. Language settings incorporate all language relevant configurations, such as time zone, keyboard layout, and system locales.
  3. The basic settings define essential properties, such as host and domain name, LDAP base, Windows domain name as well as the system administrators (root) password.
  4. In the network settings, IP addresses (IPv4 and IPv6) as well as name servers, gateways, and HTTP proxies may be specified.
  5. Following the certificate settings, a new root certificate will be created for the domain. Note that this step only applies to systems with the role 'domaincontroller master'
  6. Via the software settings, particular software components may be installed or removed.

During the saving process, a domain join will be executed, as well. For this, it is necessary to enter a valid domain administrator account on all system roles except domaincontroller master, where a new domain will be created. After the successful domain join, the configuration wizard may be closed in order to continue with the boot process.

The configuration can also be conducted via a web browser from an external system. In this case, a subsequent relogin at Univention Management Console will be necessary.

The initial configuration can be modified at any later point via the UMC module "Basic settings".
