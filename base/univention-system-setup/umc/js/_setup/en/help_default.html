UCS initial configuration

General

The UCS initial configuration allows to adapt basic settings of a Univention Corporate Server system after the boot process via a web browser. In this way, prepared images of Univention Corporate Server (UCS) can be copied or, for virtual instances, cloned. A configuration is then possible at system startup.

The basic configuration incorporates necessary system settings in order to join a domain and to be operational. Moreover, additional settings, such as software equipment, may be adjusted.

Configuration

The basic system configuration is divided into the following steps:

  1. Language settings incorporate all language relevant configurations, such as time zone, keyboard layout, and system loclaes.
  2. The basic settings define essential properties, such as host and domain name, LDAP base, Windows base as well as root password.
  3. In the network settings, IP addresses (IPv4 and IPv6) as well as name servers, gateways, and HTTP proxies may be specified.
  4. Following the certificate settings, a new root certificate will be created for the domain. Note that this step only applies to domaincontroller master systems.
  5. Via the software settigns, particular software components may be installed or removed.

During the saving process, a domain join will be executed, as well. For this, it is necessary to enter a valid domain administrator account on all system roles except domaincontroller master. After the successful domain join, the configuration wizard can be closed in order to continue with the boot process.

The configuration can also be conducted via a web browser from an external system. In this case, a subsequent relogin at Univention Management Console will be necessary.

The basic configuration can be modified at any later point via the UMC moudle "Basic settings".
